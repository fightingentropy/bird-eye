About

What this dashboard does

This UI calls the bird CLI on your machine to fetch tweets from your account’s home feed and renders them as cards in the dashboard.

How it works

Data source

The server runs the bird CLI locally and calls bird list-timeline 1933193197817135501 -n 10 --json (or your configured command). This uses your existing X/Twitter session cookies to read that list timeline.

Cache behavior

Results are cached per command. The dashboard only refreshes when you click Refresh or Apply. Every other load returns the cached payload.

Command config

You can set the command on the dashboard. For safety, the server allows only bird home and bird list-timeline with --following (home only) and -n/--count.

Allowed commands & flags

Command

  • bird list-timeline <list> — list timeline (default)
  • bird home — home timeline (For You)
  • bird home --following — following feed

Flags

  • -n <count> or --count <count> — 1 to 50
  • --json — optional; ignored if included

Examples

bird list-timeline 1933193197817135501 -n 10 --json
bird home --following -n 20
bird home -n 5
